# Artifact Policy

This repository keeps source code, schemas, task definitions, small fixtures, human baselines, and compact result summaries in Git. High-volume generated artifacts are excluded so the portfolio copy stays reviewable and does not publish raw model traces.

## Keep In Git

- `agent/` source code and tests.
- `data/schema/` canonical action schemas.
- `data/tasks/*/task.json`, `candidate_actions.json`, `notes.md`, `testcases/`, and small `human_baseline/` fixtures.
- `data/reports/` registry and collection summaries.
- `eval/` evaluator code, tests, scripts, and compact reports under `eval/results/`.

## Keep Out Of Git

- Kaggle API credentials.
- Downloaded Kaggle competition datasets.
- Notebook collection caches under `data/collector/data/`.
- Task-local raw datasets under `data/tasks/*/data/`.
- High-volume generated agent outputs under `data/tasks/*/outputs/`.
- Temporary traces, prompts, streams, and cost telemetry produced by local runs.

## Restoring Outputs

Generated task outputs can be regenerated by running the relevant agent runner against a task bundle after restoring the raw Kaggle data. If an archived artifact bundle is maintained separately, restore it into:

```text
data/tasks/<competition-slug>/outputs/
```

Then run:

```bash
uv run python eval/scripts/validate_artifacts.py
uv run python eval/aggregate.py --task <competition-slug> --stage-scope primary --format markdown --success-threshold 0.5
```
